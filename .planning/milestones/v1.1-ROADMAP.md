# Milestone v1.1: Icon Library Integration

**Status:** SHIPPED 2026-03-04
**Phases:** 9-12
**Total Plans:** 8

## Overview

Integrate established icon libraries into Motif's design system pipeline so composed screens use real, domain-appropriate icons instead of placeholders. Adds curated icon library reference, per-vertical icon vocabularies, pipeline wiring for automatic icon selection and composition, and enforcement via reviewer agent and aria-check hook.

## Milestone Summary

**Key Decisions:**
- 4 curated icon libraries: Phosphor, Lucide, Material Symbols, Tabler (Heroicons excluded — 316 icons insufficient)
- Default recommendation: Lucide (widest adoption, shadcn/ui alignment)
- Deterministic selection algorithm: vertical + personality -> library + weight (no agent guesswork)
- Icon name hallucination prevention: curated per-vertical manifest of 15-25 validated names
- CDN version pinning mandatory — never use @latest
- CSS class + CDN delivery (inline SVG is an anti-pattern for agents)
- Icon size tokens follow 8px-multiple scale: 16/20/24/32/40px
- Fixed icon size scale (not project-adjustable) — system invariant
- rem units for icon tokens — matches --space-* and --text-* convention
- Token naming --icon-{scale} (not --icon-size-{scale}) — follows --text-sm, --radius-md pattern
- ICON-CATALOG.md is REQUIRED (not optional) for screen composition
- Icon enforcement checks conditional on ICON-CATALOG.md existence — graceful skip when absent
- Direct cp sync for source-to-installed file updates

**Issues Resolved:**
- Bracket placeholder icon references replaced with vocabulary-referencing notation across all verticals
- Context engine stale vertical path fixed (verticals/ -> references/verticals/)
- Icon-only button detection handles <a> alongside <button> for same a11y concern

**Issues Deferred:**
- Dark mode icon weight adjustment (v1.2+, ICON-01)
- Phosphor duotone secondary color control (v1.2+, ICON-02)
- Material Symbols variable font axis tokens (v1.2+, ICON-03)
- Icon search/discovery tool for agents (v1.2+, ICON-04)
