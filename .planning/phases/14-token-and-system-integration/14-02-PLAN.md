Create `scripts/gap-analyzer.js` — a zero-dependency Node.js script.

**CLI interface:**
```
node scripts/gap-analyzer.js [project-root] [--vertical <name>] [--output-dir <dir>]
```
- `project-root` defaults to cwd
- `--vertical` reads from STATE.md if not provided
- `--output-dir` defaults to project-root

**Inputs:**
1. `.planning/design/PROJECT-SCAN.md` — parse the component catalog section. Extract component name, file path, type (primitive/composite/feature), and confidence from the markdown tables.

**Matching algorithm (3-tier):**

1. **Exact match (HIGH confidence):** Scanned component name matches required name (case-insensitive).

2. **Alias match (MEDIUM confidence):** Use an alias table for known equivalents:
   ```javascript
   const ALIASES = {
     table: ['datatable', 'grid', 'datagrid'],
     nav: ['navbar', 'navigation', 'sidebar', 'sidenav', 'topbar', 'header'],
     dropdown: ['select', 'menu', 'popover', 'combobox', 'listbox'],
     toast: ['notification', 'snackbar', 'alert', 'banner'],
     modal: ['dialog', 'overlay', 'popup', 'drawer', 'sheet'],
     input: ['textfield', 'textinput', 'formfield', 'textbox'],
     badge: ['chip', 'tag', 'label', 'statuschip', 'indicator'],
     avatar: ['userpic', 'profileimage', 'userimage'],
     card: ['panel', 'tile', 'container'],
     button: ['btn', 'cta', 'action'],
   };
   ```

3. **Contains match (MEDIUM confidence):** Scanned component name contains the required name or vice versa (e.g., "TransactionListRow" contains "Transaction").

Only consider scanned components with HIGH or MEDIUM confidence from PROJECT-SCAN.md. Ignore LOW-confidence components.

**Output: COMPONENT-GAP.md**
Write to `{outputDir}/.planning/design/COMPONENT-GAP.md` following the exact format from the research "COMPONENT-GAP.md Output Format" section:
- Summary counts
- Existing table (Required | Found As | File | Match Quality)
- Missing table (Component | Type | Description)
- Partial Matches table (if any MEDIUM-confidence matches)
- Token budget: target ~600 tokens, cap at 800

**Edge cases:**
- No PROJECT-SCAN.md: exit with message "No PROJECT-SCAN.md found. Run /motif:scan first."
- No component catalog in PROJECT-SCAN.md: exit with "No components found in PROJECT-SCAN.md"
- Unknown vertical: use only CORE_COMPONENTS, warn about unrecognized vertical
- No STATE.md and no --vertical flag: use only CORE_COMPONENTS, note "vertical unknown"

**CLI output:** Print summary to stdout: "X of Y required components found. Z to generate."
