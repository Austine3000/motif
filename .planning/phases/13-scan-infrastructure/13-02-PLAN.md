**Create `core/workflows/scan.md`** -- the scan workflow orchestrator instructions.

This workflow runs in the MAIN context (not a subagent) because it requires user interaction for corrections. Structure:

1. **Run scanner:** Execute `node scripts/project-scanner.js [projectRoot]` where projectRoot is the target project (from $ARGUMENTS or cwd)
2. **Read outputs:** Load the generated PROJECT-SCAN.md and CONVENTIONS.md
3. **Present summary:** Show compact overview to user:
   - Framework: [name] [version] (confidence)
   - CSS: [approaches] (confidence)
   - Components: [N] found across [M] directories
   - Conventions: [key findings]
4. **Drill-down confirmation:** For each major section, present findings with choice-based corrections per user decision:
   - Framework: "I detected [X]. Correct? [Yes / No, it's Y / Other]"
   - CSS Approach: "I detected [X]. Correct? [Yes / No, it's Y / Other]"
   - Component Catalog: Show grouped-by-directory summary. "Review the component catalog above. Any corrections? [Looks good / I want to adjust some]"
     - If adjusting: let user reclassify confidence or category for specific components
   - Conventions: Show key findings with inconsistencies. "These are the patterns I found. Any corrections? [Looks good / Override: X should be Y]"
5. **Apply corrections:** If user made corrections, update PROJECT-SCAN.md and CONVENTIONS.md accordingly
6. **Confirm:** "Scan complete. Results saved to .planning/design/PROJECT-SCAN.md and .planning/design/CONVENTIONS.md. You can manually edit these files before running /motif:system."
7. **Update STATE.md:** Add PROJECT-SCAN.md and CONVENTIONS.md to Context Budget table. Do NOT change the Phase field (scan is a pre-step, not a state change per research recommendation).

Key principle from user: "Motif discovers, the user decides, the agents execute." The scan should feel instant and informative, not a long interrogation. Present everything at once, let user correct what's wrong.

## Brownfield Detection

1. Check if `package.json` exists AND at least one source directory exists (src/, app/, lib/, pages/)

   - Skip scanning entirely (greenfield project)
   - Continue to interview as before (no behavior change)

- Pre-fill detected vertical from project type (e.g., fintech if financial dependencies detected)
- Pre-fill detected stack from framework detection
