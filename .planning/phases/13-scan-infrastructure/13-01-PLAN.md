Create `scripts/project-scanner.js` as a zero-dependency Node.js script. Follow the exact style of existing scripts (contrast-checker.js shebang, 'use strict', pure Node.js built-ins only).

**Pipeline architecture** (sequential stages):

**Stage 1 -- detectFramework(projectRoot):**
- Read package.json, merge dependencies + devDependencies
- Check meta-frameworks BEFORE base frameworks (order: next, nuxt, remix, astro, svelte, vue, react, angular)
- Return `{ name, version, confidence }`. If no package.json, return `{ name: 'unknown', confidence: 'LOW' }`
- Also detect language: check for tsconfig.json -> TypeScript, else JavaScript

**Stage 2 -- detectCSSApproach(projectRoot):**
- Check tailwind config files (tailwind.config.js/ts/mjs) AND tailwindcss in deps
- Check for .module.css/.scss/.less files (CSS Modules) -- use findFiles helper, limit search depth to 3
- Check styled-components, @emotion/react, @emotion/styled in deps
- Fallback: vanilla-css with MEDIUM confidence
- Return array of `{ name, confidence }` (projects can use multiple)

**Stage 3 -- mapDirectoryStructure(projectRoot):**
- Walk top 3 levels of source directories (src/, app/, lib/, pages/, components/)
- Count files per directory
- Return tree structure with annotations (e.g., "# Next.js App Router (12 routes)")
- Use the SKIP_DIRS set from research: node_modules, .git, dist, build, .next, .nuxt, .output, .svelte-kit, coverage, .cache, .turbo, .vercel, __pycache__, .pytest_cache, vendor, .bundle, .planning, .claude, .motif-backup

**Stage 4 -- catalogComponents(projectRoot, structure, framework):**
- Walk project files (use walkProject helper with symlink protection via lstatSync, max depth 10, skip SKIP_DIRS)
- Only scan TEXT_EXTENSIONS: .js, .jsx, .ts, .tsx, .vue, .svelte
- For each file, compute confidence score using multi-signal approach:
  - Directory heuristics: components/ui/widgets/elements/atoms/molecules (+30), pages/views/screens/routes (+20), layouts/templates (+20)
  - Export pattern: matches COMPONENT_PATTERNS for framework (+25)
  - JSX presence: `<[A-Z]` or `return\s*\(?\s*<` (+20)
  - PascalCase filename (+15)
  - Negative signals: test/spec/story files (-50), hooks/utils/helpers dirs (-20), index/types/utils basename (-20)
  - Score >= 50 = HIGH, >= 25 = MEDIUM, else LOW
- For React: use regex patterns from research (export default function, export function, export const, arrow functions, class components)
- For Vue/Svelte: .vue/.svelte files are always components
- Extract component name from filename or export pattern
- Categorize using multi-signal approach:
  - Directory: pages/ -> page, layouts/ -> layout, components/ui/ -> primitive
  - Size: < 30 lines likely primitive, > 200 lines likely page/composite
  - Import count: many component imports -> composite/page
  - Name matching: Button/Input/Badge -> primitive, Modal/Card/Dropdown -> composite, Dashboard/Settings -> page, Sidebar/Header/Footer -> layout
- For HIGH-confidence components, extract props via regex:
  - Destructured props: `function Component({ prop1, prop2 }: Props)`
  - Interface/type definition: `interface ComponentProps { prop: string; }`
  - Track prop name and required status
- Group components by directory for output
- Cap file reading at 50KB per file (skip larger files -- likely generated)

**Stage 5 -- selectRepresentativeFiles(components):**
- Filter HIGH-confidence components
- Group by directory, pick median-sized file from each (up to 3 directories)
- Fill remaining slots (up to 5 total) from largest directory
- Return selected files for convention sampling

**Stage 6 -- extractConventions(projectRoot, representativeFiles):**
- For each representative file, read content and apply CONVENTION_PATTERNS from research:
  - Styling: borderRadius (tailwind rounded-X, css border-radius, css vars), spacing (p/m/gap values), shadows (shadow-X, box-shadow), colors (bg/text/border classes, hex values, css vars)
  - Code: import style (named vs default vs barrel), export style (named vs default), naming case (kebab/pascal/camel from filenames)
- Also detect: path aliases (check tsconfig.json for paths config), barrel files (index.ts in component dirs)
- Count occurrences, compute percentages, flag inconsistencies (when dominant pattern < 70% usage)

**Output functions:**

**writeProjectScan(outputDir, scanData):**
- Write `.planning/design/PROJECT-SCAN.md` in the format from research
- Sections: Framework, CSS Approach, Directory Structure (tree), Component Catalog (grouped by directory with table), Props Summary (HIGH-confidence only)
- Budget: ~1200 tokens. Enforce by: directory-level summaries with counts, only list individual components in tables, cap tables at 20 rows per directory with "...and N more" truncation

**writeConventions(outputDir, conventions):**
- Write `.planning/design/CONVENTIONS.md` in the format from research
- Sections: Styling Conventions (border-radius, spacing, shadows, colors with percentages and inconsistency flags), Code Conventions (imports, file structure, exports)
- Budget: ~800 tokens. Enforce by: top-3 values per category, skip categories with < 2 occurrences

**CLI entry:**
- Accept projectRoot as first argument (default: process.cwd())
- Accept --output-dir as optional flag (default: projectRoot)
- Run full pipeline, print summary to stdout
- Exit 0 on success, exit 1 on error (with message)
- Wrap entire scan in try/catch -- never crash for individual file errors

**Error handling throughout:**
- All fs.readFileSync calls wrapped in try/catch (skip unreadable files silently)
- Track visited directories by realpath to prevent symlink loops
- lstatSync before entering directories to detect symlinks
- Max recursion depth of 10
