<task type="auto">
  <name>Task 2: Create validate-tokens.js -- token quality and brand color checker</name>
  <files>test/validation/validate-tokens.js</files>
  <action>
Create `test/validation/validate-tokens.js` as a CLI script that accepts a tokens.css file path and optional `--brand-color #HEXVAL` flag.

Structure:
1. Shebang, strict mode.
2. Accept `process.argv[2]` as tokensPath (required). Parse `--brand-color` flag if present.
3. Define `BANNED_FONTS = ['Inter', 'Roboto', 'Open Sans', 'Lato', 'Arial', 'Helvetica', 'system-ui']`.
4. Function `extractTokenValue(content, tokenName)` -- extracts value of a CSS custom property from tokens.css content. Pattern: `--{tokenName}:\s*([^;]+);`.
5. Function `checkBannedFonts(content)` -- extracts --font-display and --font-body values, checks against BANNED_FONTS. Returns array of issues. NOTE: If the font line contains "LOCKED" comment, skip it (user chose intentionally).
6. Function `checkPrimaryExists(content)` -- verifies --color-primary-500 token exists. Returns issue if missing.
7. Function `checkContrastAnnotations(content)` -- checks that tokens.css contains WCAG contrast annotations (look for "AA" or "contrast" or "ratio" case-insensitive). Returns issue if absent.
8. Function `hexToHsl(hex)` -- converts hex color to {h, s, l} object. Use the standard algorithm (research code example). Returns hue in degrees (0-360).
9. Function `verifyBrandColor(content, expectedHex)` -- checks that --color-primary-500 value exactly matches expectedHex (case-insensitive hex comparison). Returns `{ preserved: bool, actual, expected }`.
10. Function `checkTokenStructure(content)` -- verifies tokens.css has :root block, contains at least 20 custom properties (basic sanity), and has color/font/spacing/radius token categories.
11. Main: read tokens.css, run all checks, print results with assert pattern. If --brand-color provided, also run verifyBrandColor. Exit 0/1.

Use only Node.js built-ins. Output must match the assert/console style from e2e-installer.js.
  </action>
  <verify>Run `node -c test/validation/validate-tokens.js` -- no syntax errors. Run `node test/validation/validate-tokens.js` -- should print usage and exit 1.</verify>
  <done>Script validates tokens.css quality (banned fonts, required tokens, WCAG annotations, structure) and optionally verifies brand color preservation with exact hex match on --color-primary-500.</done>
</task>

<task type="auto">
  <name>Task 3: Create validate-diff.js -- differentiation comparison tool</name>
  <files>test/validation/validate-diff.js</files>
  <action>
Create `test/validation/validate-diff.js` as a CLI script that accepts two tokens.css file paths and reports whether they are "visibly distinct."

Structure:
1. Shebang, strict mode.
2. Accept `process.argv[2]` as tokensPathA, `process.argv[3]` as tokensPathB (both required).
3. Reuse hexToHsl from validate-tokens.js by implementing it inline (keep scripts self-contained per project zero-dependency policy).
4. Function `extractPrimaryHue(content)` -- extracts --color-primary-500 hex and converts to HSL hue. Returns degrees (0-360) or null.
5. Function `extractFont(content, tokenName)` -- extracts font family name from --font-display or --font-body. Returns the font name string (first font in the list, stripped of quotes).
6. Function `extractRadius(content)` -- extracts --radius-md value. Returns the numeric value in px.
7. Function `compareSystems(contentA, contentB)` -- returns:
   - `hueDifference`: absolute hue difference in degrees (handle wraparound: min of |diff| and 360-|diff|)
   - `fontsDifferent`: boolean (--font-display values differ)
   - `radiusDifferent`: boolean (--radius-md values differ)
   - `surfaceDifferent`: boolean (--color-surface values differ)
8. Define thresholds per research:
   - `HUE_THRESHOLD = 30` (degrees -- "visibly distinct" means hue shift >= 30)
   - fonts must be different families
   - radius must differ by at least one step
9. Function `assessDistinctness(comparison)` -- returns `{ distinct: bool, reasons: string[], failures: string[] }`. Distinct requires: hue >= 30 AND fonts different. Radius and surface are bonus checks reported but not required for pass.
10. Main: read both files, run compareSystems, run assessDistinctness, print report. Exit 0 if distinct, exit 1 if not.

Use only Node.js built-ins. Follow assert/console pattern from e2e-installer.js.
  </action>
  <verify>Run `node -c test/validation/validate-diff.js` -- no syntax errors. Run `node test/validation/validate-diff.js` -- should print usage and exit 1.</verify>
  <done>Script compares two tokens.css files and reports quantified visual distinctness: hue difference >= 30 degrees, different display fonts, different radii. Exits 0 if systems are visibly distinct, 1 if not.</done>
</task>
