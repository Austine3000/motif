  <files></files>
  <action>
The user must configure trusted publishing on npmjs.com. This cannot be automated -- it requires the npmjs.com web UI.

**Pre-requisite:** The @sailslab npm organization must exist. If not, create it at https://www.npmjs.com/org/create (free for public packages).

**Option A -- If OIDC trusted publishing can be configured before first publish:**

1. Go to https://www.npmjs.com/settings/sailslab/packages -- click "New Package" or configure at the org level
2. Set up trusted publishing:
   - Repository owner: `SailsLab`
   - Repository name: `motif`
   - Workflow filename: `publish.yml`
   - Environment: (leave blank)
3. Then trigger the first publish by creating and pushing a tag:
   ```bash
   git tag v0.1.0
   git push origin v0.1.0
   ```
4. Monitor the GitHub Actions run: `gh run watch`

**Option B -- If npm requires the package to exist first (manual first publish):**

1. Create a granular access token at https://www.npmjs.com/settings/~/tokens/granular-access-tokens/new
   - Token name: "motif-first-publish"
   - Expiration: 7 days (shortest available)
   - Packages: Read and write
   - Organizations: @sailslab
2. Run the first publish locally:
   ```bash
   npm publish --access public --token YOUR_TOKEN_HERE
   ```
3. After successful first publish, configure trusted publishing at https://www.npmjs.com/package/@sailslab/motif/access:
   - Under "Trusted Publisher", select "GitHub Actions"
   - Repository owner: `SailsLab`
   - Repository name: `motif`
   - Workflow filename: `publish.yml`
   - Environment: (leave blank)
4. Revoke the granular token immediately after configuring trusted publishing
5. Verify trusted publishing works by bumping and tagging:
   ```bash
   npm version patch
   git push origin main --follow-tags
   ```
6. Monitor: `gh run watch`
  </action>

  </verify>
