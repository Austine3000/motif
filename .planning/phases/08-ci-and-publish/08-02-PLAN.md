<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: First npm publish and trusted publisher configuration</name>
  <files>package.json</files>
  <action>
The first publish requires human interaction because npm OIDC trusted publishing may need the package to exist before configuration, and the npm account/org must be set up if not already.

**Option A: First publish with granular token (recommended for first time)**

1. Log in to npmjs.com
2. If using a scoped name like `@sailslab/motif`:
   - Ensure the "sailslab" npm org exists (create at npmjs.com/org/create if not)
3. Create a granular access token:
   - npmjs.com -> Access Tokens -> Generate New Token -> Granular Access Token
   - Permissions: Read and Write
   - Select packages: Only select packages -> the package name
   - Expiration: 7 days (only needed for first publish)
4. From the project root, run:
   ```bash
   npm config set //registry.npmjs.org/:_authToken=YOUR_TOKEN
   npm publish --access public
   npm config delete //registry.npmjs.org/:_authToken
   ```
5. Verify the package exists: `npm view <package-name>`

**Option B: Try OIDC first (if npm supports pre-publish trusted publisher config)**

1. On npmjs.com, navigate to your profile -> Packages -> look for option to pre-register a package
2. If available, configure trusted publisher:
   - Organization/user: SailsLab (or your GitHub username)
   - Repository: motif
   - Workflow filename: publish.yml
   - Environment: npm-publish
3. Then tag and push to trigger the workflow:
   ```bash
   git tag v0.1.0
   git push origin v0.1.0
   ```
4. Check workflow status: `gh run list --workflow=publish.yml`

**After first publish (either option), configure trusted publishing:**

1. Go to npmjs.com -> package page -> Settings (or Access)
2. Under "Trusted Publishers", click "Add trusted publisher"
3. Select "GitHub Actions"
4. Fill in (CASE SENSITIVE):
   - Organization or user: `SailsLab` (or your GitHub username, must match exactly)
   - Repository: `motif` (must match exactly)
   - Workflow filename: `publish.yml` (must match exactly, include .yml extension)
   - Environment: `npm-publish` (must match exactly -- this matches the `environment:` field in the workflow YAML)
5. Save

**Verify trusted publishing works by doing a patch release:**
```bash
npm version patch
git push origin main --follow-tags
```
Then check: `gh run list --workflow=publish.yml` and `npm view <package-name> versions`

Note: `npm version patch` modifies package.json (bumps version) and creates a git commit + tag automatically.
