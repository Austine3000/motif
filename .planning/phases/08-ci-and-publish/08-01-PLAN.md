  <action>
Create `.github/workflows/publish.yml` with the complete tag-triggered publish workflow:

```yaml
name: Publish to npm

on:
  push:
    tags:
      - 'v*.*.*'

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      id-token: write
    steps:
      - name: Checkout
        uses: actions/checkout@v6

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: '22.x'
          registry-url: 'https://registry.npmjs.org'

      - name: Install latest npm (for trusted publishing)
        run: npm install -g npm@latest

      - name: Verify package contents
        run: npm pack --dry-run

      - name: Verify tag matches package.json version
        run: |
          PKG_VERSION="v$(node -p "require('./package.json').version")"
          GIT_TAG="${GITHUB_REF#refs/tags/}"
          if [ "$PKG_VERSION" != "$GIT_TAG" ]; then
            echo "::error::package.json version ($PKG_VERSION) does not match git tag ($GIT_TAG)"
            exit 1
          fi

      - name: Publish to npm
        run: npm publish --provenance --access public
```

Key design decisions in the workflow:
- `id-token: write` permission enables OIDC trusted publishing (tokenless, no secrets to manage)
- `npm install -g npm@latest` ensures npm >= 11.5.1 which is required for OIDC trusted publishing (Node 22.x ships with npm 10.x which does not support it)
- `npm pack --dry-run` verifies package contents before publishing (catches accidentally included files)
- Tag-version consistency check prevents publishing with mismatched versions
- `--provenance` flag generates supply chain attestation (required by npm for trusted publishing)
- `--access public` ensures scoped package is published publicly (scoped packages default to private)
