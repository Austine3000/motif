2. **Update `package.json`** -- read the existing file first, then update:
   - Change `"name"` to the chosen package name (e.g., `"@sailslab/motif"`)
   - Add `"repository"` field: `{ "type": "git", "url": "git+https://github.com/SailsLab/motif.git" }` (adjust org/repo if user specifies differently)
   - Add `"publishConfig"` field: `{ "access": "public", "provenance": true }`
   - Keep all existing fields (version, description, bin, files, engines, license, keywords) unchanged

3. **Create `.github/workflows/publish.yml`** (create the `.github/workflows/` directory first):
   ```yaml
   name: Publish to npm

   on:
     push:
       tags:
         - 'v*.*.*'

   jobs:
     publish:
       runs-on: ubuntu-latest
       # OIDC trusted publishing requires an environment to scope the token exchange.
       # Create this environment in GitHub: Settings -> Environments -> New environment -> "npm-publish"
       # No environment protection rules needed (tag filter on workflow is sufficient).
       environment: npm-publish
       permissions:
         contents: read
         id-token: write
       steps:
         - name: Checkout
           uses: actions/checkout@v6

         - name: Setup Node.js
           uses: actions/setup-node@v6
           with:
             node-version: '22.x'
             registry-url: 'https://registry.npmjs.org'

         - name: Install latest npm (for trusted publishing)
           run: npm install -g npm@latest

         - name: Verify package contents
           run: npm pack --dry-run

         - name: Verify tag matches package.json version
           run: |
             PKG_VERSION="v$(node -p "require('./package.json').version")"
             GIT_TAG="${GITHUB_REF#refs/tags/}"
             if [ "$PKG_VERSION" != "$GIT_TAG" ]; then
               echo "::error::package.json version ($PKG_VERSION) does not match git tag ($GIT_TAG)"
               exit 1
             fi

         - name: Publish to npm
           run: npm publish --provenance --access public
           # If OIDC trusted publishing is not yet configured or fails,
           # you can fall back to a granular access token by uncommenting:
           # env:
           #   NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
   ```

   **Important:** The `environment: npm-publish` field is required for OIDC trusted publishing. It scopes the OIDC token to a named GitHub environment, which npm uses to verify the publisher identity. Without it, the OIDC token exchange may fail with an authentication error. After creating the GitHub repo, create this environment at: Settings -> Environments -> New environment -> name it "npm-publish".
