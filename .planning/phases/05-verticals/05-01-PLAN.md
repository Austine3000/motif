   - **LogEntry** (name MUST be exactly "LogEntry"): Single health event in timeline (meal, workout, symptom). Include structure, dimensions, states (default/hover/editing), tap-target
8. **Interaction Patterns:**
   - Core Flows: Logging a health metric, viewing trend data, setting a health goal
   - States: Loading (skeleton shimmer matching expected content density), Empty (encouraging first-time messages), Error (reassuring tone, never alarming about data loss)
   - Motion: Appropriate (progress ring fill animation 600ms, metric count-up 400ms, sparkline draw 500ms) and Inappropriate (bouncy/playful springs, fast aggressive transitions, gamification effects)
9. **Accessibility Specifics:** Health-specific -- larger min font for older users (16px body minimum), color-blind safe metric ranges (not color alone), screen reader announces metric values with units and trend, medication reminders must support visual + audio + haptic
10. **Border Radius:** Token table (sm=6px, md=12px, lg=16px, xl=20px) -- friendlier/more rounded than fintech to feel less clinical

12. **Health-Specific Additions:** Medication reminders UX, health metric normal range indicators (green/yellow/red bands), HIPAA-awareness in data display (no PHI in notifications), date/time formatting for medical events, wearable data sync indicators
