<task type="auto">
  <name>Task 2: Build bin/install.js -- complete fresh-install pipeline</name>
  <files>bin/install.js</files>
  <action>
Create bin/install.js as a single-file synchronous pipeline using ONLY Node.js 22+ stdlib (zero npm dependencies). The file must start with `#!/usr/bin/env node` shebang and `'use strict';`.

**Required imports (all `node:` prefix):**
- `node:fs` -- readFileSync, writeFileSync, existsSync, mkdirSync, readdirSync, copyFileSync, unlinkSync, renameSync, statSync
- `node:path` -- join, resolve, relative, dirname, basename, extname, sep
- `node:util` -- parseArgs, styleText
- `node:crypto` -- createHash

**Stage 1: parseFlags()**
Use `util.parseArgs` with strict mode. Support these flags:
- `--runtime` / `-r` (string) -- override auto-detection
- `--force` / `-f` (boolean, default false) -- skip backup, overwrite all
- `--dry-run` (boolean, default false) -- print actions without writing
- `--uninstall` (boolean, default false) -- remove installation
- `--help` / `-h` (boolean, default false) -- print usage

If `--help`, print a usage message showing all flags and exit 0.
If `--uninstall`, call uninstall() and exit (implemented in Plan 02 -- for now, print "Uninstall not yet implemented" and exit 1).

**Stage 2: detectRuntime(flags)**
If `--runtime` flag provided, validate it is in `['claude-code']` (v1.0 only). If invalid, print error with supported runtimes and exit 1.
If no flag, auto-detect: check if `.claude/` exists in cwd. If found, return `'claude-code'`.
If no runtime detected, print error suggesting `--runtime claude-code` and exit 1.

**Stage 3: resolveMapping(runtime)**
Return an object with:
- `forgeRoot`: `'.claude/get-motif'`
- `copies`: array of `{ src, dest }` pairs:
  - `core/references` -> `.claude/get-motif/references`
  - `core/workflows` -> `.claude/get-motif/workflows`
  - `core/templates` -> `.claude/get-motif/templates`
  - `runtimes/claude-code/agents` -> `.claude/get-motif/agents`
  - `runtimes/claude-code/commands/forge` -> `.claude/commands/motif` (note: forge -> motif directory rename)
- `snippet`: path to `runtimes/claude-code/CLAUDE-MD-SNIPPET.md`
- `configTarget`: `path.join(cwd, 'CLAUDE.md')`

Use `path.dirname(__dirname)` for the npm package root (parent of bin/).
Skip any source directory that does not exist (graceful handling for scripts/, hooks/ not yet built).

**Stage 4: copyFiles(mapping, flags)**
For each entry in `mapping.copies`:
1. Check if `src` directory exists. If not, skip with a note to console.
2. Call `walkAndCopy(src, dest, forgeRoot, flags)` which:
   - Uses `readdirSync(dir, { withFileTypes: true })` to walk entries
   - Skips `.DS_Store` and dotfiles (entry.name starts with '.')
   - Creates dest directory with `mkdirSync(dest, { recursive: true })`
   - For directories: recurse
   - For files: validate resolved path is within `process.cwd()` (path traversal check)
   - For `.md` files: read content, run `resolveContent(content, forgeRoot)`, write resolved content
   - For non-`.md` files: use `copyFileSync` verbatim
   - If `--dry-run`: print "Would copy: {relPath}" and skip actual write
   - Track `{ copied, skipped }` counts

The `resolveContent(content, forgeRoot)` function does:
```javascript
return content
  .replaceAll('{FORGE_ROOT}', forgeRoot)
  .replaceAll('.claude/get-design-forge', forgeRoot);
```
This resolves both the `{FORGE_ROOT}` variable AND the hardcoded old path references.

**Stage 5: injectConfig(mapping, flags)**
Read the snippet from `mapping.snippet`. Wrap it: `<!-- MOTIF-START -->\n{snippet}\n<!-- MOTIF-END -->`.
Handle three cases:
1. CLAUDE.md exists and has sentinel markers: replace content between markers with new block
2. CLAUDE.md exists but no markers: append block with `\n\n` separator
3. CLAUDE.md does not exist: check `.claude/CLAUDE.md` -- if exists, use that path instead. If neither exists, create at `mapping.configTarget`.
If `--dry-run`: print what would happen and skip actual write.
Return `{ action: 'replaced' | 'appended' | 'created', path: resolvedPath }`.

**Stage 6: hashFile(filePath) helper**
```javascript
function hashFile(filePath) {
  const content = fs.readFileSync(filePath);
  return createHash('sha256').update(content).digest('hex');
}
```

**Stage 7: writeManifest(mapping, copyResults, flags)**
Write `.motif-manifest.json` at project root with:
```json
{
  "version": "0.1.0",
  "runtime": "claude-code",
  "installedAt": "ISO-timestamp",
  "files": { "relative/path": { "hash": "sha256-hex", "source": "relative-source-path" } }
}
```
Build the files object by walking all installed files (from the mapping.copies dest directories + the config target) and computing hashes. If `--dry-run`, skip writing.

**Stage 8: verify(mapping)**
Check for errors:
1. Each expected directory from `mapping.copies` (dest) must exist
2. Grep all `.md` files under `.claude/get-motif/` and `.claude/commands/motif/` for `{FORGE_ROOT}` -- any match is an error
3. CLAUDE.md (or `.claude/CLAUDE.md`) must contain `<!-- MOTIF-START -->` and `<!-- MOTIF-END -->`
4. `.motif-manifest.json` must exist
Return array of error strings (empty = success).

**Stage 9: printSummary(copyResult, injectResult, verifyErrors)**
Use `util.styleText` for colored output:
- Bold header: "Motif Installation Summary"
- Separator line: '─'.repeat(40)
- Files copied count, config action (created/appended/replaced)
- If verifyErrors is empty: green "Installation verified successfully." + get-started hints (/motif:init, /motif:help)
- If verifyErrors exist: red error list and `process.exit(1)`

**Main flow (bottom of file):**
```javascript
const flags = parseFlags();
const runtime = detectRuntime(flags);
const mapping = resolveMapping(runtime);
const copyResult = copyFiles(mapping, flags);
const injectResult = injectConfig(mapping, flags);
if (!flags['dry-run']) {
  writeManifest(mapping, copyResult, flags);
}
const verifyErrors = flags['dry-run'] ? [] : verify(mapping);
printSummary(copyResult, injectResult, verifyErrors);
```

**Security:** All target paths must be validated with `path.resolve()` + `startsWith(process.cwd() + path.sep)` check before any write operation. Exit 1 immediately on path traversal detection.

**Error handling:** Wrap individual file operations in try/catch. Accumulate errors but continue copying. Print failed files in summary. Exit 1 if any operations failed.

The file should be approximately 300-400 lines total. Do NOT split into modules.
