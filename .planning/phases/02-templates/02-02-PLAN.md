Create `core/templates/token-showcase-template.html` -- a polished, self-contained HTML page that visually renders all design tokens from a sibling `tokens.css` file.

**Structure requirements** (derived from generate-system.md "Output 4" lines 302-311 and token file format lines 151-251):

1. **Head section:**
   - `<!DOCTYPE html>`, `<html lang="en">`, charset UTF-8, viewport meta
   - `<title>{PRODUCT_NAME} -- Design Tokens</title>`
   - `<link rel="stylesheet" href="tokens.css">` (relative sibling import)
   - `<link href="https://fonts.googleapis.com/css2?family={FONT_FAMILIES}" rel="stylesheet">` (placeholder for Google Fonts)
   - All presentation CSS in a single inline `<style>` block (self-contained requirement)

2. **Inline CSS must include:**
   - Reset: `*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }`
   - Body using token variables: `font-family: var(--font-body)`, `background: var(--surface-primary)`, `color: var(--text-primary)`, `padding: var(--space-8)`, `line-height: var(--leading-normal)`
   - Headings using `var(--font-display)`
   - Section layout: max-width container (~1200px), centered, sections with vertical spacing
   - `.swatch` class: 80x80px boxes with `border-radius: var(--radius-md)`, `border: 1px solid var(--border-primary)`
   - `.swatch-label` class: `font-family: var(--font-mono)`, `font-size: var(--text-xs)`
   - Swatch grid layout: CSS Grid or Flexbox, wrapping rows of color swatches
   - Typography preview styles for rendering each scale step at its actual size
   - Spacing visualization: horizontal bars showing relative spacing values
   - Radius preview: boxes with each radius applied
   - Shadow preview: boxes with each shadow applied
   - Component preview containers
   - A fallback warning if tokens.css is missing: `body::before` content rule that shows "tokens.css not found" -- hidden when custom properties resolve
   - Responsive: single column on mobile, multi-column on desktop

3. **Body sections** (in order, matching generate-system.md Output 4):

   **Header:**
   - `<h1>{PRODUCT_NAME} Design Tokens</h1>`
   - `<p>Vertical: {VERTICAL} | Generated: {ISO_DATE}</p>`

   **Section: Colors (`id="colors"`)**
   - `<h2>Color Palette</h2>`
   - Subsections with HTML comments marking where the agent fills in swatches:
     - Primary scale (50-950): grid of `.swatch` divs, each with `background-color: var(--color-primary-{step})` and a label showing the token name + hex value placeholder
     - Semantic colors: success, error, warning, info swatches
     - Surface colors: primary, secondary, tertiary, elevated
     - Text colors: primary, secondary, tertiary, inverse, link (show as text samples on appropriate backgrounds)
     - Border colors: primary, focus

   **Section: Typography (`id="typography"`)**
   - `<h2>Typography Scale</h2>`
   - Font family display: show `--font-display`, `--font-body`, `--font-mono` names with sample text
   - Each scale step (text-xs through text-4xl) rendered at its actual size using `font-size: var(--text-{size})`
   - Weight samples: normal (400), medium (500), semibold (600), bold (700)

   **Section: Spacing (`id="spacing"`)**
   - `<h2>Spacing Scale</h2>`
   - Visual bars: for each spacing token (space-1 through space-16), render a horizontal bar with `width: var(--space-{N})` and a label showing the token name and computed value placeholder

   **Section: Border Radius (`id="radii"`)**
   - `<h2>Border Radius</h2>`
   - Row of boxes, each with a different `border-radius: var(--radius-{size})` applied, labeled with the token name

   **Section: Shadows (`id="shadows"`)**
   - `<h2>Shadows</h2>`
   - Row of boxes, each with a different `box-shadow: var(--shadow-{size})` applied, labeled

   **Section: Component Previews (`id="components"`)**
   - `<h2>Component Previews</h2>`
   - Placeholder containers for Button (primary, secondary, ghost variants), Input, Card, and Badge components
   - Use token variables for all styling (colors, fonts, spacing, radii, shadows)
   - The system architect agent fills these with actual styled HTML during generation

   **Section: Vertical-Specific (`id="vertical-specific"`)**
   - `<h2>Vertical-Specific Tokens</h2>`
   - HTML comment: `<!-- Agent fills in tokens unique to the detected vertical -->`
   - Empty placeholder section that the agent populates based on the vertical

4. **Placeholder conventions:**
   - `{PRODUCT_NAME}`, `{VERTICAL}`, `{ISO_DATE}`, `{FONT_FAMILIES}` -- replaced by the system architect agent
   - CSS `var(--token-name)` references -- resolved by the browser from tokens.css
   - HTML comments (`<!-- ... -->`) mark where the agent adds dynamic content

5. **Constraints:**
   - NO JavaScript (not even inline `<script>` tags)
   - NO external dependencies except tokens.css and Google Fonts CDN
   - ALL layout/presentation CSS in the single inline `<style>` block
   - File must be a valid, complete HTML document that renders correctly when opened in a browser
   - The page should look polished and professional -- this is a design system showcase, not a raw debug page

6. **Anti-patterns to avoid:**
   - Do NOT hardcode specific hex color values -- always use `var(--token-name)` so the showcase reflects whatever tokens are generated
   - Do NOT use JavaScript for any dynamic rendering
   - Do NOT use any CSS framework (Tailwind, Bootstrap, etc.)
   - Do NOT add `<script>` tags
