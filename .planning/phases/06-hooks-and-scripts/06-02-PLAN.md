Create the statusLine script following 06-RESEARCH.md Pattern 2. This is NOT a PostToolUse hook -- it is a statusLine command that receives session JSON on stdin and prints formatted text.

**motif-context-monitor.js (HOOK-04):**
- Shebang: `#!/usr/bin/env node` with `'use strict';`
- Read JSON from stdin using async `process.stdin.on('data'/'end')` pattern
- Parse session JSON: extract `data.context_window.used_percentage` (default to 0 if missing)
- Calculate percentage as integer: `Math.floor(pct)`
- Color-code output using ANSI escape sequences:
  - 0-29%: GREEN (`\x1b[32m`) -- "Motif context: NN%"
  - 30-49%: GREEN -- same format
  - 50-69%: YELLOW (`\x1b[33m`) -- "Motif context: NN% -- recommend /clear"
  - 70-89%: YELLOW with stronger warning -- "Motif context: NN% -- quality may degrade, /clear recommended"
  - 90%+: RED (`\x1b[31m`) -- "Motif context: NN% -- CRITICAL: /clear now"
- Reset ANSI at end: `\x1b[0m`
- Output via `process.stdout.write()` (NOT console.log -- avoid extra newline that could affect status line display)
- Keep this script fast: no file I/O, no git commands, no network calls. Just parse JSON and print.
- Wrap JSON.parse in try/catch -- if stdin is malformed, exit 0 silently (never crash the status line)
  </action>
  <verify>

```bash
echo '{"context_window":{"used_percentage":35}}' | node runtimes/claude-code/hooks/motif-context-monitor.js

# Warning threshold (50%+)

```

Each test should show colored output at correct thresholds. Malformed input should exit 0 silently.

<task type="auto">
  <name>Task 2: Create contrast-checker and token-counter utility scripts (SCRP-01, SCRP-02)</name>
  <files>scripts/contrast-checker.js, scripts/token-counter.js</files>
  <action>
Create two utility scripts in scripts/. Both are pure Node.js, zero external dependencies, meant to be run directly from the command line by agents or users.

**contrast-checker.js (SCRP-01):**
- Shebang: `#!/usr/bin/env node` with `'use strict';`
- CLI interface: `node scripts/contrast-checker.js <color1> <color2>` where colors are hex values (with or without #)
- Parse hex colors: support both 3-char (#abc) and 6-char (#abcdef) formats. Expand 3-char to 6-char.
- sRGB linearization using WCAG 2.1 threshold (0.04045, NOT the old 0.03928):
  ```
  function linearize(channel) {
    const sRGB = channel / 255;
    return sRGB <= 0.04045 ? sRGB / 12.92 : Math.pow((sRGB + 0.055) / 1.055, 2.4);
  }
  ```
- Relative luminance: `L = 0.2126 * R_lin + 0.7152 * G_lin + 0.0722 * B_lin`
- Contrast ratio: `(lighter + 0.05) / (darker + 0.05)` where lighter = max(L1, L2), darker = min(L1, L2)
- Output format:
  ```
  Colors: #AAAAAA vs #FFFFFF
  Contrast ratio: 2.32:1
  WCAG AA normal text (4.5:1): FAIL
  WCAG AA large text (3:1):   FAIL
  WCAG AAA normal text (7:1): FAIL
  WCAG AAA large text (4.5:1): FAIL
  ```
- If fewer than 2 args, print usage and exit 1
- If color format invalid, print error and exit 1
- Known test values for verification: black (#000000) vs white (#FFFFFF) = 21:1, pure red (#FF0000) vs white = ~4:1

**token-counter.js (SCRP-02):**
- Shebang: `#!/usr/bin/env node` with `'use strict';`
- CLI interface: `node scripts/token-counter.js [directory]` -- defaults to `.planning/design/` if no argument
- Use `fs` and `path` from Node.js built-ins
- Walk the target directory recursively, reading all files
- Skip binary files (check for null bytes in first 512 bytes) and .DS_Store
- Count total characters across all text files
- Approximate token count: `Math.ceil(totalChars / 4)` (standard ~4 chars/token heuristic)
- Output format:
  ```
  Directory: .planning/design/
  Files scanned: 12
  Total characters: 48,320
  Approximate tokens: ~12,080
  ```
- If directory doesn't exist, print message and exit 1
- Format large numbers with commas for readability
