Create two PostToolUse hook scripts following the exact pattern from 06-RESEARCH.md Pattern 1.

**motif-token-check.js (HOOK-01):**
- Shebang: `#!/usr/bin/env node` with `'use strict';`
- Read JSON from stdin using async `process.stdin.on('data'/'end')` pattern
- Extract file_path from `data.tool_input.file_path`
- Early exit (exit 0, no output) if file extension is NOT one of: css, tsx, jsx, vue, html
- Get content to check: `data.tool_input.content` for Write, `data.tool_input.new_string` for Edit (per Research Pattern 4)
- Skip lines inside comments: lines starting with `//` or `/*` through `*/` blocks and lines starting with `*` (inside block comments). Also skip HTML comments `<!-- -->`.
- Check for violations using these regex patterns (match property context, not bare values):
  - Hardcoded colors: `/(color|background(?:-color)?|border(?:-color)?|outline-color|fill|stroke)\s*:\s*(#[0-9a-fA-F]{3,8}|rgba?\(|hsla?\()/g` -- but ALLOW if the value uses `var(`
  - Hardcoded spacing: `/(margin|padding)(?:-(top|right|bottom|left))?\s*:\s*\d+px/g` and `/gap\s*:\s*\d+px/g` -- but ALLOW `0px` and `1px` (common resets)
  - Hardcoded font-size: `/font-size\s*:\s*\d+(?:px|rem|em)/g`
  - Hardcoded border-radius: `/border-radius\s*:\s*\d+px/g` -- ALLOW `0px` (reset) and `50%`/`9999px` (circles)
  - Hardcoded box-shadow: `/box-shadow\s*:\s*(?!var\(|none)/g`
- For each violation, collect: the matched text, line number, and a suggestion (e.g., "Use var(--color-primary) or appropriate token from tokens.css")
- If violations found, output JSON: `{ "decision": "block", "reason": "Motif token-check: N violation(s) found:\n- Line X: ... Use var(--token-name) instead.\n..." }`
- If no violations, exit 0 with no output (silent pass)
- Do NOT read files from disk -- only inspect content/new_string from stdin JSON

**motif-font-check.js (HOOK-02):**
- Same shebang and stdin reading pattern as token-check
- Same file extension filter (css, tsx, jsx, vue, html)
- Same Write vs Edit content extraction
- Banned fonts list (all lowercase for comparison): `inter`, `roboto`, `open sans`, `opensans`, `lato`, `arial`, `helvetica`, `system-ui`, `-apple-system`, `blinkmacsystemfont`, `segoe ui`
- Before checking, attempt to read user-locked fonts from tokens.css. Use `$CLAUDE_PROJECT_DIR` env var to locate it: `path.join(process.env.CLAUDE_PROJECT_DIR || '.', '.planning', 'design', 'system', 'tokens.css')`. If tokens.css exists, parse `--font-display` and `--font-body` token values. Extract the font name from these values (e.g., `--font-display: "Roboto", sans-serif` means "roboto" is user-locked). Wrap in try/catch -- if tokens.css doesn't exist or can't be read, proceed with empty locked list.
- Match `/font-family\s*:\s*([^;}\n]+)/gi` in content
- For each match, skip if value contains `var(` (already tokenized)
- For each match, check if any banned font name appears in the value (case-insensitive). Skip if that font is in the user-locked list.
- Collect violations with line number, the banned font found, and suggestion: "Use var(--font-display) or var(--font-body) instead"
- Output decision JSON if violations found, silent exit if clean

  <name>Task 2: Create a11y-check hook (HOOK-03)</name>

  <action>
Create the accessibility check PostToolUse hook.

**motif-aria-check.js (HOOK-03):**
- Same shebang, stdin reading, file extension filter, Write/Edit extraction as the other hooks
- Target file extensions: tsx, jsx, vue, html (NOT css -- CSS has no HTML elements)
- Three violation checks:

1. **div with onClick but missing role/tabIndex:**
   - Match `/<div[^>]*\bonClick\b[^>]*>/gi`
   - For each match, check if it contains BOTH `role=` AND `tabIndex=` (case-insensitive for tabindex)
   - If either is missing, it's a violation
   - Suggestion: "Add role=\"button\" and tabIndex={0} to make this div keyboard-accessible, or use a <button> element instead"

2. **img without alt:**
   - Match `/<img\b[^>]*>/gi`
   - For each match, check if it contains `alt=`
   - Missing alt is a violation (even `alt=""` is acceptable -- it means decorative)
   - Suggestion: "Add alt=\"description\" to this img, or alt=\"\" if decorative"

3. **input without label association:**
   - Match `/<input\b[^>]*>/gi`
   - For each match, check if it has `aria-label=` OR `aria-labelledby=` OR `id=` (which could be paired with a label for=)
   - If none present, it's a violation
   - Suggestion: "Add aria-label=\"description\" or ensure a <label htmlFor=\"id\"> exists"
   - Note: inputs with `type="hidden"` should be skipped (they don't need labels)

- Collect all violations with line number, violation type, matched element snippet (truncated to 80 chars), and suggestion
- Output decision JSON if any violations found
- Format reason clearly: "Motif a11y-check: N accessibility violation(s):\n- Line X: [type] ... Suggestion\n"
  </action>
